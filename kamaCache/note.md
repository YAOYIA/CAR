# 什么是缓存
缓存是将高频访问的数据暂存到内存中，是加速数据访问的存储，降低延迟，提高吞吐率的利器。
# 为什么要使用缓存
通过牺牲服务器的一部分空间，减少对磁盘或者数据库资源的直接读写，可以换取更加快的响应速度。
# 缓存策略对比
|方法|优势|缺点|应用场景|
|---|---|---|---|
LRU(least recently used)|1,实现方法简单，基于时间自然访问淘汰策略；2.适合短期热点数据，命中率高；3.易于其缓存机制结构结合|循环缓存问题：对于扫描型的访问，可能会频繁淘汰热点数据，导致缓存抖动|适合访问的模式：短期热点突出的场景
|---|---|---|
|LFU(least frequenty used)|更关注访问频率，能够保留长期使用的重要数据；2.对稳定的热点数据命中率更高|冷启动问题：新数据可能尚未证明重要性前被淘汰；数据频率的更新有额外的开销；不适合访问模式变化频繁的场景|适合访问叛逆率稳定，长期热点数据占主导的场景|
